Fluxo de acesso (manual)

O diagrama resume o caminho típico após abrir o site. Texto alternativo: ver lista numerada abaixo.

flowchart TD A[Abrir Politapp] --> B{Autenticado?} B -->|Năo| C["login.html — Google"] C --> D{Conta aprovada?} D -->|Pendente| E[aguarde-aprovacao.html] D -->|Recusada| F[conta-recusada.html] D -->|OK / admin| G["Demais páginas"] B -->|Sim| G G --> H["Tarefas, Insights, Mídia…"]
  1. Sem sessăo, o auth-guard envia para o login (exceto páginas públicas como landing-publico.html).
  2. Primeiro acesso cria perfil; administradores podem aprovar ou recusar.
  3. Com sessăo válida, o menu leva a todas as áreas; dados sensíveis do WhatsApp ficam no browser até exportar.

Dados da Câmara e perfil de candidato

  • Lista (index): paginaçăo, filtro por UF, ligaçăo ao detalhe do deputado.
  • Perfil (candidato.html): abre a partir da lista; traz biografia, comissőes e contexto da API.
  • API oficial: link «Documentaçăo da API» no menu — útil para cruzar campos avançados.

Território RJ — vereadores e insights

  • Vereadores RJ: na mesma família de páginas, use o sub-menu (lista, Caxias, mapa) para năo se perder.
  • Insights RJ: tręs blocos principais — mapa de necessidades por município, comércio/feiras com horários de referęncia, viagem OSRM + POIs.
  • Prefeituras: visăo municipal complementar ao mapa estadual.

Transparęncia e mídia social

  • Transparęncia: painéis e referęncias para acompanhamento de informaçăo pública.
  • Mídia social: escuta curada + integraçőes; para APIs com token use o servidor de desenvolvimento e proxy conforme README do projeto.

WhatsApp e integraçőes

  • whatsapp.html: preencha por blocos (provedor, credenciais, webhook, comportamento do bot), depois Salvar.
  • Use Exportar JSON para partilhar configuraçăo com o backend sem copiar tokens ŕ măo em chats inseguros.
  • Em produçăo, migre credenciais para variáveis de ambiente no servidor.

Conta, tarefas e administraçăo

  • Conta: estado do utilizador e mensagens de configuraçăo (Supabase).
  • Tarefas: escolha data, unidade (se aplicável), marque itens; notas longas no mesmo ecră.
  • Admin (admin.html): apenas perfil admin — unidades e aprovaçőes de novos utilizadores.

Se algo năo carregar, confirme ligaçăo ŕ rede, ficheiro js/auth-config.js e consola do navegador (F12).
