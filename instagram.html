Instagram Graph API Bot & agendamento

Painel para configurar integração Instagram com a Meta (conta comercial/creator ligada a uma Página Facebook): publicação de feed, fila programada no seu backend e opcionalmente automação de DMs onde as permissões do app permitirem. Credenciais ficam só neste navegador (localStorage) até exportar — em produção use servidor + cofre.

Guia de implementação · Painel de configuração

Guia de implementação

A Instagram Content Publishing API cria contenções de mídia e publica no feed/reels conforme tipo e permissões. A Meta não agenda horários por si: o seu serviço (cron, Cloud Scheduler, fila Bull, etc.) chama a API no momento combinado.

Pré-requisitos

  1. Conta Instagram Profissional (Empresa ou Creator) ligada a uma Página do Facebook que você gere.
  2. App em Meta for Developers com Instagram Graph API / produtos corretos e app em modo Live quando for escala real.
  3. Token de acesso com escopos adequados (ex.: instagram_basic, pages_show_list, instagram_content_publish; para mensagens, fluxos e revisão adicionais da Meta).

Fluxo de publicação (feed — imagem)

  1. POST /{ig-user-id}/media com image_url (URL pública HTTPS, requisitos da documentação) e caption — resposta traz id do content creation.
  2. Consulte o estado do container até estar pronto (campo status_code no objeto).
  3. POST /{ig-user-id}/media_publish com creation_id para publicar.

Exemplo mínimo (valores ilustrativos):

curl -X POST "https://graph.facebook.com/v21.0/<IG_USER_ID>/media" \
  -d "image_url=https://exemplo.com/carteira.jpg" \
  -d "caption=Texto%20do%20post%20%23politica" \
  -d "access_token=<PAGE_ACCESS_TOKEN>"

curl -X POST "https://graph.facebook.com/v21.0/<IG_USER_ID>/media_publish" \
  -d "creation_id=<CONTAINER_ID>" \
  -d "access_token=<PAGE_ACCESS_TOKEN>"

Vídeo, carrossel e Reels têm parâmetros e limites próprios — siga a documentação atual da Meta para cada tipo.

Postagens programadas

  1. Guarde na base de dados ou fila: horário UTC/local, legenda, URL da mídia (ou caminho no storage), tipo de post e idempotência (ex.: slug do conteúdo).
  2. Um worker acordado pelo agendador executa o fluxo container → publish; em caso de falha, re-tentativas com backoff e alerta.
  3. Respeite limites de taxa da API e políticas comerciais; em campanhas, alinhe com legislação eleitoral e regras de anúncio.

Bot e DMs

Respostas automáticas em mensagens diretas costumam exigir Instagram Messaging, webhook e permissões em revisão na Meta. Trate o bot como um serviço à parte: recebe eventos, aplica regras (como nas palavras-chave abaixo) e chama os endpoints de envio permitidos ao seu app.

Webhook

O mesmo modelo de verificação hub.verify_token / hub.challenge (GET) e eventos (POST) usado no ecossistema Meta aplica-se aqui; subscreva os campos necessários (ex.: comentários, mensagens) no painel do app.

Sessão e persistência

Use Salvar após preencher cada bloco. Limpar sessão remove só os dados desta página neste dispositivo.

1 · Modelo e stack

Integração direta com a Instagram Graph API (recomendado) ou camada intermediária (SaaS / gateway) sob contrato.

2 · Credenciais do app e da página (nunca em repositório público)

Tokens permitem publicar e ler dados da conta. Gire segredos se vazarem. Use token de Página de longa duração no servidor que publica.
